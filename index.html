Semantic HTML

Using appropriate semantic elements helps screen readers understand the page structure and provide a better experience for users.

Semantic HTML example

Good example:

<header>
  <h1>Website Title</h1>
  <nav>
    <ul>
      <li><a href="#home">Home</a></li>
      <li><a href="#about">About</a></li>
    </ul>
  </nav>
</header>

<main>
  <article>
    <h2>Article Title</h2>
    <p>Article content...</p>
  </article>
</main>

<footer>
  <p>Copyright © 2023</p>
</footer>

Bad example:

<div id="header">
  <div class="title">Website Title</div>
  <div class="nav">
    <div><a href="#home">Home</a></div>
    <div><a href="#about">About</a></div>
  </div>
</div>

<div id="content">
  <div class="post">
    <div class="post-title">Article Title</div>
    <div class="post-content">Article content...</div>
  </div>
</div>

<div id="footer">
  <div>Copyright © 2023</div>
</div>

Tip:

Use semantic elements like <header>, <nav>, <main>, <article>, <section>, and <footer> to improve accessibility.

Images & alt text

All meaningful images should include alt text describing the image for users who cannot see it.

Alt text for images

Good example:

<img src="cat.jpg" alt="White cat playing with a red yarn ball">

This alt text accurately describes the image.

Bad example:

<img src="cat.jpg" alt="Cat">

This alt text is not sufficient and is too generic.

For purely decorative images:

<img src="divider.png" alt="">

Use an empty alt attribute for decorative images that convey no information.

Forms & accessibility

Forms should be usable with keyboard only and include clear labels.

Accessible form

Good example:

We will not share your email with anyone.
Preferred contact method:

Form tips:

  • Associate each input with a label using the for and id attributes
  • Use fieldset and legend for groups of options
  • Provide clear, accessible error messages
  • Ensure forms are usable with keyboard only

Navigation & accessibility

Navigation should be easy to use for all users, including keyboard users and screen reader users.

Navigation links

Good example:

  • Home
  • Services
  • About
  • Contact

Navigation tips:

  • Provide "skip to main content" links to allow users to bypass long menus
  • Use aria-current="page" to indicate the current page
  • Ensure tab order is logical and predictable
  • Provide visual focus styles for links

    • Tables & accessibility

    Tables should have clear headers and relationships between headers and cells to help screen readers understand the data.

    Accessible table

    Good example:

    Product Prices
    Product Price Available Quantity
    Tablet 1200 SAR 15
    Smartphone 800 SAR 22
    Laptop 2500 SAR 8

    Table tips:

    • Use <caption> to provide a title for the table
    • Use <th scope="col"> for column headers and <th scope="row"> for row headers
    • Use <thead>, <tbody>, and <tfoot> to group sections
    • Avoid very complex multi-level tables when possible

    ARIA attributes

    ARIA (Accessible Rich Internet Applications) attributes provide additional accessibility when HTML alone is insufficient.

    Common ARIA uses

    Example of a toggle button:

    <button 
  aria-expanded="false" 
  aria-controls="dropdown-menu">
  Menu
</button>
<ul id="dropdown-menu" hidden>
  <li><a href="#item1">Item 1</a></li>
  <li><a href="#item2">Item 2</a></li>
</ul>

    Example of an alert message:

    <div 
  role="alert" 
  aria-live="assertive">
  Changes saved successfully
</div>

    Important rule:

    Do not use ARIA when a native semantic HTML element already provides the behavior. Use ARIA only when HTML lacks the needed semantics.

    Accessibility testing

    After applying accessibility principles, it is important to test your site to ensure it is actually accessible.

    Ways to test accessibility:

    • Use your site with keyboard only (no mouse)
    • Test with a screen reader such as NVDA or VoiceOver
    • Use automated evaluation tools like WAVE or axe
    • Test with real users with disabilities
    • Test with different zoom levels
    • Test with high contrast settings
